¿Qué es Node.js + Express?

🔷 ¿Qué es Node.js?

Node.js es un entorno de ejecución de JavaScript del lado del servidor. Te permite ejecutar código JavaScript fuera del navegador, por ejemplo, en un servidor.

  • Permite crear servidores web.
  • Es rápido y eficiente gracias a su motor V8 (de Google Chrome).
  • Ideal para aplicaciones en tiempo real como chats, juegos, etc.
¿Para qué se usa?
Para crear APIs, servidores backend, automatizar tareas, etc.

🔷 ¿Qué es Express?

Express es un framework minimalista para Node.js que facilita la creación de servidores y APIs. Simplifica mucho el trabajo de configurar rutas, respuestas, errores, etc.

  • Fácil de usar y entender.
  • Permite crear APIs REST de manera muy rápida.
  • Se utiliza junto con Node.js para manejar rutas, peticiones, y respuestas.
¿Para qué se usa?
Para crear backends rápidos, APIs RESTful y manejar peticiones HTTP fácilmente.

🔗 ¿Cómo trabajan juntos?

Node.js ejecuta el servidor, mientras que Express organiza la lógica (rutas, respuestas, errores). Express se instala como un paquete dentro de un proyecto Node.js.

Instalación:

# Instalar Node.js desde nodejs.org (una vez)
# Luego crea tu proyecto:
npm init -y
npm install express

Ejemplo de servidor con Node.js + Express:

// index.js
const express = require('express');
const app = express();
const port = 3000;

// Ruta principal
app.get('/', (req, res) => {
  res.send('¡Hola, mundo desde Express!');
});

// Servidor en marcha
app.listen(port, () => {
  console.log(`Servidor escuchando en http://localhost:${port}`);
});
💡 Si visitas http://localhost:3000 verás el mensaje "¡Hola, mundo desde Express!".

⚙️ Cosas importantes a tener en cuenta

  • Necesitas tener Node.js instalado.
  • Debes iniciar el servidor cada vez que lo edites con: node index.js.
  • Puedes usar herramientas como nodemon para reiniciar el servidor automáticamente.
  • Express es muy usado junto a bases de datos como MongoDB o MySQL.
